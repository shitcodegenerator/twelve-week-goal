# 0) 作品題目：團購主訂單系統（可多團購主使用）

  

## 0.1 系統目標（職缺語言）


- **Multi-tenant SaaS**：支援多個團購主（tenants）共用同一套系統、資料隔離（data isolation）
- **Public ordering**：消費者可下單（無登入或輕量登入）

- **Admin console**：團購主後台管理商品/訂單/出貨/對帳

- **LINE Integration**：串接 LINE Messaging API / Webhook，推播訂單狀態、提醒團購主、通知群組/會員（以官方帳號為入口）

- **Scalability & reliability**：可水平擴充（horizontal scaling）、高併發保護（rate limiting / caching / async jobs）

- **Security**：AuthN/AuthZ、OWASP 基礎面、secrets 管理

- **Observability**：結構化 log、錯誤追蹤、基本 metrics 概念

  

## 0.2 核心角色（Roles）

  

- **Customer**（外部下單者）

- **Host**（團購主 / 店主）

- **Staff**（團購主團隊成員，可選）

- **Admin**（平台管理者，可選）

  

## 0.3 核心資料模型（PostgreSQL 方向）

  

- Tenant（團購主/店家）

- Product / Variant / Inventory（可後續）

- Campaign（團購檔期：開始/結束、運費規則、取貨方式）

- Order / OrderItem（含狀態機）

- Customer（可選：電話/Email/LINE userId 綁定）

- Payment（可先做「現金/轉帳」狀態，之後串金流）

- Shipment / Pickup（出貨/自取）

- LineChannel（每個 tenant 的 LINE OA 設定：channel secret/token）

  

> **Multi-tenant 資料隔離策略建議（12 週可落地版）**：

> 以 **shared database + shared schema + tenant_id**（row-level isolation）為主；每張核心表都帶 `tenant_id` 並在 service layer 做強制 scope。後續可談 RLS（Row Level Security）或 schema-per-tenant。

- cache-aside（campaign/product 公開讀）
