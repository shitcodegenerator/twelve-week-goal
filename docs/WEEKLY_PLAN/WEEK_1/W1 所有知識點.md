# 1) 產品與架構層（SaaS / Multi-tenant）

## Multi-tenant（多租戶）核心名詞

- **Multi-tenant SaaS**
    
- **Tenant（租戶）/ Tenant onboarding（租戶上線流程）**
    
- **Data isolation（資料隔離）**
    
- **Row-level isolation（列層級隔離）**
    
- **Shared database / Shared schema / Tenant ID strategy**（共用 DB + 共用 schema + `tenant_id`）
    
- **Schema-per-tenant**（每租戶一個 schema）
    
- **Database-per-tenant**（每租戶一個資料庫）
    
- **Tenant routing / Tenant resolution**（如何判定當前 tenant）
    
    - **Path-based tenant**：`/t/[tenantSlug]/...`
        
    - **Subdomain-based tenant**：`{tenant}.example.com`
        
- **Tenant scope / Tenant scoping（租戶範圍強制）**
    
- **Cross-tenant data leak（跨租戶資料外洩）**
    
- **Authorization boundary（授權邊界）**
    
- **IDOR (Insecure Direct Object Reference)**（越權讀取常見型態，Week1 會先認識概念）
    
- **Threat model（威脅模型）**（通常在 ADR/安全章節會提到）
    
- **Separation of concerns（關注點分離）**（為什麼要拆 lib/domain/components）
    

## 作品的路由/模組分區（你會做的結構）

- **Public surface area（對外公開面）**：`/t/[tenantSlug]/...`
    
- **Host admin console（後台）**：`/host/...`
    
- **BFF（Backend for Frontend）**：先認識概念（App Router 內 server/client 分工會碰到）
    

## 文件化（可交付/可說服）

- **README v0（Quick Start / Scripts / Routes）**
    
- **ADR (Architecture Decision Record)**
    
    - Context / Decision / Consequences
        
    - Alternatives considered（替代方案）
        
    - Risks / Mitigations（風險/對策）
        
- **Trade-off（取捨）**：例如 path vs subdomain、shared schema vs schema-per-tenant
    
- **Definition of Done（完成定義）**：每一天的驗收點

# 3) React（元件、資料流、Hooks、狀態管理心智）

## React 核心名詞

- **Component（元件）**
    
- **Props（屬性/輸入）**
    
- **State（狀態）**
    
- **Event handler（事件處理）**
    
- **Conditional rendering（條件渲染）**
    
- **List rendering（列表渲染）**
    
- **Key（列表 key 的意義）**
    
- **Unidirectional data flow（單向資料流）**
    
- **Lifting state up（狀態上提）**
    

## Hooks（Week1 會實際用到的）

- **useState**
    
    - Controlled inputs（受控表單）
        
    - 狀態更新模式（functional update）
        
- **useEffect**
    
    - Dependency array（依賴陣列）
        
    - Cleanup function（清理：debounce/訂閱/timeout）
        
    - 常見坑：依賴漏寫/多寫、無限迴圈
        
- **useMemo**
    
    - Memoization（記憶化）
        
    - 避免 expensive compute（例如 filter/sort）
        
    - 代價：複雜度、依賴錯會出 bug
        
- **useCallback**
    
    - Stable function reference（穩定函式引用）
        
    - 避免子元件因 prop handler 變動而 rerender
        
- （選用）**React.memo**
    
    - Shallow compare（淺比較）
        
    - 什麼時候有用、什麼時候是過早優化
        

## 狀態管理（Week1 的“入門但要說得出口”）

- **Local state（單一元件狀態）**
    
- **Shared state（跨元件共享狀態）**
    
- **Props drilling（層層傳遞）**
    
- **Derived state（衍生狀態）**
    
    - “不要把可推導的東西塞進 state” 的原則
        
- **Single source of truth（單一真相來源）**
    
- **State normalization（狀態正規化）**（Week1 先認識詞）
    
- **When to introduce state library（何時需要狀態庫）**
    
    - signals：跨頁、跨深度、多寫入者、同步困難、複雜互動
        
- **React Context（最小全域狀態示範）**
    
    - Provider / Consumer（或 `useContext`）
        
    - rerender cost（更新造成的重繪範圍）
        

## 元件工程化（你會實作的元件型態）

- **Presentational vs Container（展示/容器分離）**
    
- **Component boundary（元件邊界）**
    
- **Reusable component（可重用元件）**
    
- **Generic component（泛型元件：DataTable<T>）**
    
- **Form patterns（表單模式）**
    
    - Controlled component
        
    - Validation state（Week1 可先做最小）
